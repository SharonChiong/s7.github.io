《漫画算法》学习笔记(5)：面试中的算法

发表于 2020-08-16 更新于 2020-12-22 分类于 算法 阅读次数： 评论数：

判断链表有环

思路：创建两个指针 p1 和 p2，同时指向链表的头节点；循环链表，p1 每次向后移动1个节点，p2 每次向后移动2个节点，然后比较两个指针指向的节点是否相同。

时间复杂度：O(n); 空间复杂度：O(1).

问题扩展

  1. 如何求环的长度？

    让两个指针从相遇点继续循环前进，并统计前进的循环次数，直到两个指针第2次相遇。统计出的前进次数就是环长。

    环长 = 每一次速度差 * 前进次数 = 前进次数

  2. 如何求出入环节点？

    2(D+S1) = D+S1+S2+S1 = D+2S1+S2 => D = S2，故从链表头节点到入环点的距离，等于从首次相遇点到入环点的距离。

    只要把其中一个指针放回头节点位置，另一个指针保持在首次相遇点，两个指针都是每次向前走1步，它们最后相遇的节点就是入环节点。

最小栈的实现

进栈、出栈、取最小值的时间复杂度都是 O(1), 最坏空间复杂度为 O(n).

思路：创建一个额外的“备胎”栈，存储栈中曾经的最小值。

求出最大公约数(gcd)

暴力枚举法

时间复杂度：O(min(a,b))

辗转相除法

又欧几里得算法：两个正整数a和b(a>b)，它们的最大公约数等于a除以b的余数c和b之间的最大公约数。

存在的问题：当两个整数较大时，取模算法性能会比较差。

时间复杂度：O(log(max(a,b)))

更相减损术

a-b的差值c和较小树b的最大公约数。

算法性能不稳定，最坏时间复杂度：O(max(a,b))

更相减损术与移位相结合

当a和b均为偶数时，gcd(a,b) = 2 x gcd(a/2,b/2) = 2 x gcd(a>>1,b>>1)

当a为偶数，b为奇数时，gcd(a,b) = gcd(a/2,b) = gcd(a>>2,b)

当a和b均为奇数时，先用更相减损术运算一次，gcd(a,b) = gcd(b,a-b), 此时a-b为偶数，继续进行移位运算。

判断整数奇偶性的方式：让整数和1进行与运算，如果 (a&1)==0, 则说明整数a是偶数；反之为奇数。

算法性能稳定，时间复杂度：O(log(max(a,b)))

判断一个数是否为2的整数次幂

时间复杂度:O(logn)

移位的性能比乘法高得多。

时间复杂度:O(1)

若一个整数是2的整数次幂，当它转化成二进制时，只有最高位是1，其他位都是0. 2的整数次幂减1，它的二进制都变成了1.

无序数组排序后的最大相邻差

解法1

  1. 利用计数排序，先求出 max 和 min 的区间长度 k: k=(max-min+1), 以及偏移量 d=min;
  2. 创建一个长度 k 的新数组 Array;
  3. 遍历原数组，每遍历一个元素，就把 Array 对应下标的值+1;
  4. 遍历 Array, 统计出 Array 中最大连续出现0值的次数+1.

解法2

  1. 利用桶排序，根据原数组长度 n, 创建出 n 个桶。区间跨度：(max-min)/(n-1);
  2. 遍历原数组，把元素插入对应的桶，记录每个桶的最大和最小值；
  3. 遍历所有桶，统计出每个桶的最大值，和右侧非空桶的最小值的差。

时间复杂度: O(n)

用栈实现队列

思路：用两个栈实现一个队列。入队时，每一个新元素被压入到栈A中，出队时，让栈A中所有元素按顺序出栈，再按照出栈顺序压入栈B.

入队时间复杂度：O(1), 出队时间复杂度（均摊时间复杂度）：O(1).

寻找全排列的下一个数

思路：尽量保持高位不变，低位在最小的范围内变换顺序。变换顺序的范围大小，取决于当前整数的逆序区域。

步骤：

  1. 从后向前查看逆序区域，找到逆序区域的前1位，即数字置换的边界；
  2. 让逆序区域的前1位和逆序区域中大于它的最小的数字交换位置；
  3. 把原来的逆序区域转为顺序状态。

字典序算法，时间复杂度：O(n).

删除k个数字后的最小值

思路：把原整数的所有数字从左到右进行比较，若发现某一位数字大于它右边的数字，那么删除该数字后，必然会使该数位的值降低。

贪心算法：依次求得局部最优解，最终得到全局最优解。

以上代码的时间复杂度：O(kn); substring 方法本身性能不高, 时间复杂度是 O(n).

更优解：利用栈来回溯遍历过的数字即删除数字，时间复杂度和空间复杂度都是 O(n).

实现大整数相加

用数组逆序存储两个大整数，数组长度等于大整数位数+1，按位相加，最后把数组再次逆序并转成String.

以上代码的时间复杂度：O(n). 优化：把大整数的每9位作为数组的一个元素，进行加法运算。

金矿问题

典型的“动态规划”问题：确定全局最优解和最优子结构之间的关系，以及问题的边界。状态转移方程式。

金矿数量设为 n, 工人数量设为 w, 金矿的含金量设为数组 g[], 金矿所需开采人数设为数组 p[], 设 F(n,w) 为 n 个金矿，w 个工人时的最优收益函数。

F(n,w) = 0(n=0或w=0) 问题边界；

F(n,w) = F(n-1,w)(n>=1, w<p[n-1]) 当所剩工人不够挖掘当前金矿，只有一种最优子结构；

F(n,w) = max(F(n-1,w), F(n-1,w-p[n-1])+g[n-1])(n>=1, w>=p[n-1])) 常规情况下具有两种最优子结构。

自底向上求解。

时间复杂度：O(nw), 空间复杂度：O(nw).

优化：最优结果，是由它的两个最优子结构推导而来。两个最优子结构都位于它的上一行。

空间复杂度：O(w)
