《漫画算法》学习笔记(4) —— 排序算法

发表于 2020-08-15 更新于 2020-08-30 分类于 算法 阅读次数： 评论数：

  1. O(n^2): 冒泡排序、选择排序、插入排序、希尔排序(性能略优于 O(n^2), 又比不上 O(nlogn))

  2. O(nlogn): 快速排序、归并排序、堆排序

  3. O(n): 计数排序、桶排序、基数排序

  4. 其他：鸡尾酒排序、猴子排序、睡眠排序

根据稳定性可分为稳定排序和非稳定排序.

冒泡排序(bubble sort)

一种稳定排序。

优化：利用布尔变量 isSorted 作为标记；对数列有序区的界定。

鸡尾酒排序(cocktail sort)

基于冒泡排序的一种升级排序法。冒泡算法是从左到右来比较元素，进行单向的位置交换。

鸡尾酒算法的元素比较和交换都是双向的。使用场景是大部分元素已经有序的情况。

快速排序(quick sort)

和冒泡排序一样，属于交换排序。思路是分治法。不稳定排序。

快排的时间复杂度为 O(nlogn), 最坏时间复杂度是 O(n^2); 空间复杂度为 O(logn).

基准元素(pivot)

随机选择一个元素作为基准元素，并且让其和数列首元素交换位置。

元素交换

双向循环法

  1. 选定基准元素 pivot;

  2. 第1次循环：

    1. 从 right 指针开始，若指针所指向的元素大于等于 pivot, 指针向左移动，若小于 pivot, 指针停止移动，切换到 left 指针；

    2. 若 left 指针指向的元素小于等于 pivot，则指针向右移动，若大于 pivot, 指针停止移动；

    3. 让 left 和 right 指针指向的元素交换位置。

  3. 第2次循环：重新切换到right指针…

单向循环法

  1. 选定基准元素 pivot，同时设置一个 mark 指针指向数列起始位置，它代表小于基准元素的区域边界；

  2. 从 pivot 下一个位置开始遍历，若遍历到的元素小于基准元素：

    1. mark 指针右移一位；

    2. 让最新遍历的元素和 mark 指针指向的元素交换位置；

    3. pivot 元素交换到 mark 指向的位置。

堆排序(heap sort)

不稳定排序。

  1. 把无序数组构建成二叉堆，需要从小到大排序，则构建最大堆，反之构建最小堆；

  2. 循环删除栈顶元素，替换到二叉堆的末尾，调整堆产生新的堆顶。

空间复杂度：O(1); 时间复杂度：O(nlogn)

计数排序(count sort)

数组中每一个下标位置的值代表数列中对应整数出现的次数。适用于一定范围内的整数排序。以数列最大值-最小值+1作为统计数组的长度。

优化版的计数排序是稳定排序。思路：从统计数组的第二个元素开始，每个元素都加上前面所有元素之和。让统计数组存储的元素值，等于相应整数的最终排序位置的序号。再创建输出数组 sortedArray, 长度和输入数列一致。然后从后向前遍历输入数列。

时间复杂度：O(n+m); 空间复杂度: O(m).

局限性：当数列最大和最小值差距过大时不适合；当数列元素不是整数时也不适合。

桶排序(bucket sort)

创建的桶数量等于原始数列的元素数量，除最后一个桶只包含数列最大值外，前面各个桶的区间按照比例确定：区间跨度 = (最大值-最小值) / (桶的数量-1)

时间/空间复杂度: O(n)；极端情况下时间复杂度会退化为 O(nlogn)

小结

排序算法｜平均时间复杂度｜最坏时间复杂度｜空间复杂度｜是否稳定排序

冒泡排序｜O(n^2)|O(n^2)|O(1)|稳定

鸡尾酒排序｜O(n^2)|O(n^2)|O(1)|稳定

快速排序｜O(nlogn)|O(n^2)|O(logn)|不稳定

堆排序|O(nlogn)|O(nlogn)|O(1)|不稳定

计数排序|O(n+m)|O(n+m)|O(m)|稳定

桶排序|O(n)|O(nlogn)|O(n)|稳定
