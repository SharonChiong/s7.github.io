《漫画算法》学习笔记(2)：数据结构基础

发表于 2020-08-10 更新于 2020-12-22 分类于 算法 阅读次数： 评论数：

数组(array)

有限个相同类型的变量组成的有序集合。它的一个特点：在内存中 顺序存储。

读取/更新元素

随机读取：根据下标读取元素

时间复杂度：O(1)

插入/删除元素

插入元素操作的3种情况：

  1. 尾部插入
  2. 中间插入
  3. 超范围插入：扩容

时间复杂度：O(n)

删除元素的取巧方式：把最后一个元素复制到要删除元素的位置，然后再删除最后一个元素。

优劣势

优势：非常高的随机访问能力。有一种高效查找元素的算法叫 二分查找。
劣势：插入、删除元素会导致大量元素被迫移动。

数组适合 读操作多、写操作少。

链表(linked list)

  1. 单向链表：每个节点包含两部分，一部分是存放数据的变量 data, 一部分是指向下一节点的指针 next.
  2. 双向链表：除了 data 和 next, 还有指向前置节点的指针 prev.

链表在内存中的存储方式是 随机存储。

查找/更新节点

查找链表节点的时间复杂度：O(n)

更新节点的时间复杂度：O(1)

插入节点/删除元素

同样，有3种情况：

  1. 尾部插入/删除
  2. 头部插入/删除
  3. 中间插入/删除

时间复杂度：O(1)

栈和队列

逻辑结构：线性结构（顺序表、栈、队列）、非线性结构（树、图）
物理结构：顺序存储结构（数组）、链式存储结构（链表）

栈(stack)

先入后出(FILO, First In Last Out), 栈底(bottom), 栈顶(top).

入栈(push)和出栈(pop)的时间复杂度是 O(1).

队列(queue)

先入先出(FIFO, First In First Out), 队头(front), 队尾(rear).

用数组实现的队列可以采用 循环队列 的方式来维持队列容量的恒定。

循环列表：当 (队尾下标+1) % (数组长度) = 队头下标 时，队列就已经满了。

注意：队尾指针指向的位置永远空出1位，所以队列最大容量比数组长度小1.

入队(enqueue)和出队(dequeue)的时间复杂度是 O(1).

应用

栈：对“历史”的回溯。场景：实现递归的逻辑；面包屑导航。
队列：对“历史”的回放。场景：在多线程中，争夺公平锁的等待队列；网络爬虫。

双端队列(deque)：既可以先入先出，也可以先入后出。

优先队列：谁的优先级最高，谁先出队。基于二叉堆来实现的。

散列表

也叫作哈希表(hash table). 提供了键(Key)和值(Value)的映射关系。时间复杂度接近于 O(1).

哈希函数的实现：取模运算，`index = HashCode(Key) % Array.length. Java 中采用位运算来优化性能。

科普一下：取模运算中商遵循最可能小原则；取余运算中商遵循靠近0原则。c=a/b; r=a-(c*b)

读写(get/put)操作3

哈希冲突：不同的 key 通过哈希函数获得的下标相同。解决方法：开放寻址法；链表法。

扩容(resize)

衡量 HashMap 需要扩容的条件：HashMap.Size >= Capacity * LoadFactor

扩容步骤：1. 扩容；2. 重新 Hash.
